## Prerequisites

Before you start, make sure you have:

- **Node.js 18+** and **npm** installed
- **Python 3.11+** installed
- A **Supabase** project with the schema applied (see [Database Setup](/setup/database))
- A **Groq API key** (for the AI service)
- OAuth credentials (optional) for Google and GitHub login

<Note>
  Both the Node.js and Python backends must run simultaneously. Node.js is the system orchestrator; Python is the AI brain.
</Note>

## Setup steps

<Steps>
  <Step title="Clone the repository">
    ```bash
    git clone https://github.com/HecRodCode/kairo-integrative-project-turing.git
    cd kairo-integrative-project-turing
    ```
  </Step>

  <Step title="Configure backend-node/.env">
    Create a `.env` file in the `backend-node/` directory:

    ```env backend-node/.env
    # SERVER CONFIGURATION
    PORT=3000
    NODE_ENV=development

    # SECURITY & SESSIONS
    SESSION_SECRET=your_session_secret

    # DATABASE (POSTGRESQL - SUPABASE)
    DB_HOST=your_host
    DB_PORT=5432
    DB_NAME=your_name
    DB_USER=postgres.your_user
    DB_PASSWORD=your_password
    DATABASE_URL=your_database_url

    # SUPABASE CONFIG
    SUPABASE_URL=your_supabase_url
    SUPABASE_ANON_KEY=your_anon_key

    # FRONTEND CONNECTION
    # In production: set this to the full origin, e.g. https://hecrodcode.github.io
    FRONTEND_URL=http://127.0.0.1:5500/frontend

    # PYTHON CONNECTION
    PYTHON_API_URL=http://localhost:8000

    # SOCIAL AUTH - GITHUB (optional)
    GITHUB_CLIENT_ID=your_client
    GITHUB_CLIENT_SECRET=your_client_secret

    # SOCIAL AUTH - GOOGLE (optional)
    GOOGLE_CLIENT_ID=your_client
    GOOGLE_CLIENT_SECRET=your_client_secret

    # SMTP (for OTP email verification)
    SMTP_HOST=smtp.gmail.com
    SMTP_PORT=465
    SMTP_SECURE=true
    SMTP_USER=your_email
    SMTP_PASS=your_app_password
    SMTP_FROM="Kairo <your_email>"

    # API KEY AI
    RIWI_IA_KEY=your_ia_key
    ```

    <Warning>
      `FRONTEND_URL` must exactly match the origin where your frontend is hosted. A mismatch will cause all API requests to be blocked by CORS.
    </Warning>
  </Step>

  <Step title="Configure backend-python/.env">
    Create a `.env` file in the `backend-python/` directory:

    ```env backend-python/.env
    # PYTHON CONFIGURATION
    PORT=8000
    ENV=development

    # AI MODEL
    OPENAI_API_KEY=your_groq_api_key
    MODEL_NAME=llama-3.3-70b-versatile

    # SUPABASE CONFIG
    SUPABASE_URL=your_supabase_url
    SUPABASE_SERVICE_KEY=your_service_role_key

    # BACKEND NODE PORT
    NODE_URL=http://localhost:3000

    # FRONTEND PORT
    FRONTEND_URL=http://localhost:5500
    ```

    <Note>
      `SUPABASE_SERVICE_KEY` is the **service role** key (not the anon key). It bypasses Row Level Security so the AI service can read all coder data for plan generation.
    </Note>
  </Step>

  <Step title="Start Terminal A — Node.js orchestrator">
    In your first terminal window:

    ```bash
    cd backend-node
    npm install
    npm run dev
    ```

    Look for this confirmation message:

    ```
    🚀 KAIRO API GATEWAY STARTED SUCCESSFULLY
    ```
  </Step>

  <Step title="Start Terminal B — Python AI service">
    In a second terminal window:

    <CodeGroup>
      ```bash Linux / macOS
      cd backend-python
      python3 -m venv env
      source env/bin/activate
      pip install -r requirements.txt
      uvicorn main:app --host 0.0.0.0 --port 8000 --reload
      ```

      ```powershell Windows
      cd backend-python
      python -m venv env
      env\Scripts\Activate.ps1
      pip install -r requirements.txt
      uvicorn main:app --host 0.0.0.0 --port 8000 --reload
      ```
    </CodeGroup>

    Look for this confirmation in the logs:

    ```
    INFO: Supabase client initialized
    INFO: Uvicorn running on http://0.0.0.0:8000
    ```
  </Step>

  <Step title="Start Terminal C — Frontend">
    In a third terminal window, serve the frontend on port 5500:

    <CodeGroup>
      ```bash npx serve
      npx serve -p 5500
      ```

      ```text VS Code Live Server
      Open the frontend/ folder in VS Code
      Right-click index.html → "Open with Live Server"
      Default port: 5500
      ```
    </CodeGroup>
  </Step>

  <Step title="Verify all services are running">
    Check the health of each service:

    | Service | URL | Expected response |
    |---------|-----|-------------------|
    | Node.js API | `http://localhost:3000/api/ai/health` | `{"status": "online"}` |
    | Python AI | `http://localhost:8000/health` | `{"status": "online", "service": "Kairo AI Engine"}` |
    | Python Docs | `http://localhost:8000/docs` | FastAPI Swagger UI |
    | Frontend | `http://127.0.0.1:5500/frontend` | Kairo login page |

    <Warning>
      If the Node.js health check returns a `503`, the Python service is not reachable. Make sure both `PYTHON_API_URL` in `backend-node/.env` and the uvicorn port match.
    </Warning>
  </Step>

  <Step title="Create your first account">
    1. Open `http://127.0.0.1:5500/frontend/src/views/auth/` in your browser
    2. Click **Register** and fill in your details
    3. Select a role: **Coder** or **Team Leader**
    4. Check your email for the OTP verification code
    5. Enter the OTP to activate your account
    6. Log in — you'll be redirected to the role-appropriate dashboard

    <Tip>
      First-time Coder logins are redirected to the onboarding assessment before the main dashboard. Complete the soft skills questionnaire to unlock AI plan generation.
    </Tip>
  </Step>
</Steps>
