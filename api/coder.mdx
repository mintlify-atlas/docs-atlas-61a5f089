<Info>
  All endpoints in this group require authentication and the `coder` role. Additionally, all routes enforce the `checkOnboarding` middleware — coders who have not completed onboarding will receive a `403` response with `requiresOnboarding: true` until they call `PATCH /api/auth/complete-onboarding`.
</Info>

## GET /api/coder/dashboard

<Info>Requires authentication — role: `coder`</Info>

Returns all data needed to render the coder dashboard in a single request: user profile, soft skills assessment, Moodle progress, active AI plan summary, TL feedback notifications, risk flags, and recent performance tests.

**Response — 200 OK**

<ResponseField name="user" type="object | null">
  Profile and module data for the authenticated coder.
  <Expandable title="properties">
    <ResponseField name="fullName" type="string">Display name.</ResponseField>
    <ResponseField name="email" type="string">Email address.</ResponseField>
    <ResponseField name="clanId" type="string | null">Assigned clan.</ResponseField>
    <ResponseField name="role" type="string">`coder`</ResponseField>
    <ResponseField name="kairoScore" type="number">Current Kairo score (defaults to 50 for new users).</ResponseField>
    <ResponseField name="firstLogin" type="boolean">Onboarding status.</ResponseField>
    <ResponseField name="moduleId" type="number | null">Current module ID.</ResponseField>
    <ResponseField name="moduleName" type="string | null">Current module name.</ResponseField>
    <ResponseField name="moduleTotalWeeks" type="number | null">Total weeks in the current module.</ResponseField>
    <ResponseField name="isModuleCritical" type="boolean | null">Whether the current module is flagged as critical.</ResponseField>
    <ResponseField name="learningStyleCache" type="string | null">Cached learning style label.</ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="softSkills" type="object | null">
  Latest soft skills assessment scores.
  <Expandable title="properties">
    <ResponseField name="autonomy" type="number">Score 1–5.</ResponseField>
    <ResponseField name="timeManagement" type="number">Score 1–5.</ResponseField>
    <ResponseField name="problemSolving" type="number">Score 1–5.</ResponseField>
    <ResponseField name="communication" type="number">Score 1–5.</ResponseField>
    <ResponseField name="teamwork" type="number">Score 1–5.</ResponseField>
    <ResponseField name="learningStyle" type="string">Detected learning style.</ResponseField>
    <ResponseField name="assessedAt" type="string">ISO 8601 timestamp of the assessment.</ResponseField>
    <ResponseField name="average" type="number">Average across all five dimensions.</ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="progress" type="object">
  Moodle learning progress.
  <Expandable title="properties">
    <ResponseField name="currentWeek" type="number">Active week number (defaults to 1).</ResponseField>
    <ResponseField name="averageScore" type="number | null">Average Moodle score; `null` means no data yet.</ResponseField>
    <ResponseField name="strugglingTopics" type="array">Array of topic identifiers the coder is struggling with.</ResponseField>
    <ResponseField name="weeksCompleted" type="array">Array of completed week records.</ResponseField>
    <ResponseField name="weeksCompletedCount" type="number">Total number of completed weeks.</ResponseField>
    <ResponseField name="updatedAt" type="string | null">ISO 8601 timestamp of last progress update.</ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="activePlan" type="object | null">
  Summary of the coder's current active AI-generated complementary plan.
  <Expandable title="properties">
    <ResponseField name="id" type="number">Plan ID.</ResponseField>
    <ResponseField name="targetedSoftSkill" type="string">Soft skill the plan focuses on.</ResponseField>
    <ResponseField name="planType" type="string">Plan type from the AI (e.g., `interpretive`).</ResponseField>
    <ResponseField name="generatedAt" type="string">ISO 8601 timestamp.</ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="notifications" type="object">
  Recent TL feedback messages.
  <Expandable title="properties">
    <ResponseField name="unread" type="number">Count of unread feedback items.</ResponseField>
    <ResponseField name="items" type="array">Up to 5 most recent feedback items, each with `id`, `text`, `type`, `tlName`, `createdAt`, and `isRead`.</ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="riskFlags" type="array">
  Up to 3 active (unresolved) risk flags. Each item has `level`, `reason`, and `detectedAt`.
</ResponseField>

<ResponseField name="performanceTests" type="array">
  Up to 3 most recent performance test results. Each item has `score`, `status`, `moduleName`, and `takenAt`.
</ResponseField>

**Example**

```bash
curl http://localhost:3000/api/coder/dashboard -b cookies.txt
```

## GET /api/coder/plan

<Info>Requires authentication — role: `coder`</Info>

Returns the coder's current active complementary plan with full content and day-completion state.

**Response — 200 OK**

<ResponseField name="hasPlan" type="boolean">
  `false` when no active plan exists.
</ResponseField>

<ResponseField name="plan" type="object | null">
  Full plan data when `hasPlan` is `true`.
  <Expandable title="properties">
    <ResponseField name="id" type="number">Plan ID.</ResponseField>
    <ResponseField name="moduleId" type="number">Module the plan targets.</ResponseField>
    <ResponseField name="planContent" type="object">Full plan content as generated by the AI.</ResponseField>
    <ResponseField name="softSkillsSnapshot" type="object">Soft skills data captured at plan generation time.</ResponseField>
    <ResponseField name="moodleStatusSnapshot" type="object">Moodle progress snapshot at plan generation time.</ResponseField>
    <ResponseField name="targetedSoftSkill" type="string">Primary soft skill the plan addresses.</ResponseField>
    <ResponseField name="generatedAt" type="string">ISO 8601 generation timestamp.</ResponseField>
    <ResponseField name="completedDays" type="object">Map of day number → `{ completedAt: ISO string }` for completed days.</ResponseField>
    <ResponseField name="currentDay" type="number">First day not yet completed (1–20).</ResponseField>
    <ResponseField name="completedCount" type="number">Total days completed.</ResponseField>
    <ResponseField name="isComplete" type="boolean">`true` when all 20 days are completed.</ResponseField>
  </Expandable>
</ResponseField>

**Example**

```bash
curl http://localhost:3000/api/coder/plan -b cookies.txt
```

## POST /api/coder/plan/request

<Info>Requires authentication — role: `coder`</Info>

Triggers generation of a new AI complementary plan. The Node.js server responds immediately with a `requested: true` acknowledgement and then fires a background request to the Python AI service. Poll `GET /api/coder/plan` after a few seconds to retrieve the result.

**Request body**

<ParamField body="plan_type" type="string">
  Plan generation strategy. Defaults to `interpretive` if not provided.
</ParamField>

<ParamField body="current_week" type="number">
  Current week number override. Defaults to `1` if not provided.
</ParamField>

**Response — 200 OK**

<ResponseField name="requested" type="boolean">
  Always `true`.
</ResponseField>

<ResponseField name="message" type="string">
  `"Plan en generación. Reintenta en unos segundos."`
</ResponseField>

**Example**

```bash
curl -X POST http://localhost:3000/api/coder/plan/request \
  -H "Content-Type: application/json" \
  -b cookies.txt \
  -d '{"plan_type": "interpretive", "current_week": 3}'
```

## POST /api/coder/plan/:planId/day/:day/complete

<Info>Requires authentication — role: `coder`</Info>

Marks a day (1–20) of the specified plan as complete. Awards Kairo points for the day. If all 20 days are completed, a bonus is also awarded.

**Path parameters**

<ParamField path="planId" type="number" required>
  ID of the complementary plan.
</ParamField>

<ParamField path="day" type="number" required>
  Day number to mark as complete. Must be between 1 and 20.
</ParamField>

**Response — 200 OK**

<ResponseField name="success" type="boolean">
  `true` on success.
</ResponseField>

<ResponseField name="completedDays" type="object">
  Updated map of all completed days.
</ResponseField>

<ResponseField name="completedCount" type="number">
  Total days completed after this action.
</ResponseField>

<ResponseField name="nextDay" type="number | null">
  Next incomplete day number, or `null` if the plan is finished.
</ResponseField>

<ResponseField name="isComplete" type="boolean">
  `true` when all 20 days are done.
</ResponseField>

<ResponseField name="kairoScore" type="number">
  Updated Kairo score after point awards.
</ResponseField>

**Error responses**

| Status | Condition |
|--------|-----------|
| `400 Bad Request` | `day` is not between 1 and 20 |
| `404 Not Found` | Plan not found or not active for this coder |

**Example**

```bash
curl -X POST http://localhost:3000/api/coder/plan/42/day/3/complete \
  -b cookies.txt
```

## GET /api/coder/resource/:id/download

<Info>Requires authentication — role: `coder`</Info>

Generates a signed Supabase Storage URL to download a specific resource PDF. The signed URL is valid for **1 hour**.

**Path parameters**

<ParamField path="id" type="number" required>
  ID of the resource to download.
</ParamField>

**Response — 200 OK**

<ResponseField name="url" type="string">
  Signed download URL (expires in 3600 seconds).
</ResponseField>

<ResponseField name="fileName" type="string">
  Original file name of the PDF.
</ResponseField>

**Error responses**

| Status | Condition |
|--------|-----------|
| `404 Not Found` | Resource not found or not in the coder's clan |
| `500 Internal Server Error` | Failed to generate the signed URL |

**Example**

```bash
curl http://localhost:3000/api/coder/resource/12/download -b cookies.txt
```

## POST /api/coder/resources/search

<Info>Requires authentication — role: `coder`</Info>

Searches for resources relevant to a given topic. First attempts semantic search via the Python RAG service; falls back to PostgreSQL full-text search if the Python service is unavailable.

**Request body**

<ParamField body="topic" type="string" required>
  Topic or keyword to search for.
</ParamField>

<ParamField body="moduleId" type="number">
  Narrow results to a specific module.
</ParamField>

<ParamField body="limit" type="number">
  Maximum number of results to return. Defaults to `3`.
</ParamField>

**Response — 200 OK**

<ResponseField name="success" type="boolean">
  Always `true` (errors return empty results, not a failure status).
</ResponseField>

<ResponseField name="resources" type="array">
  Matched resources, each including `id`, `title`, `fileName`, `previewText`, `storagePath`, `moduleId`, `uploadedAt`, `similarity`, and a `download_url` (signed URL, valid 1 hour).
</ResponseField>

**Example**

```bash
curl -X POST http://localhost:3000/api/coder/resources/search \
  -H "Content-Type: application/json" \
  -b cookies.txt \
  -d '{"topic": "async/await", "moduleId": 4, "limit": 5}'
```

## PATCH /api/coder/activities/:id/complete

<Info>Requires authentication — role: `coder`</Info>

Marks a plan activity as complete and records an optional reflection and time spent. Uses upsert semantics — calling it again for the same activity updates the existing record.

**Path parameters**

<ParamField path="id" type="number" required>
  ID of the activity to mark as complete.
</ParamField>

**Request body**

<ParamField body="reflectionText" type="string">
  Optional free-text reflection on the activity.
</ParamField>

<ParamField body="timeSpent" type="number">
  Time spent on the activity in minutes.
</ParamField>

**Response — 200 OK**

<ResponseField name="success" type="boolean">
  `true` on success.
</ResponseField>

<ResponseField name="data" type="object">
  The created or updated `activity_progress` record.
</ResponseField>

**Example**

```bash
curl -X PATCH http://localhost:3000/api/coder/activities/99/complete \
  -H "Content-Type: application/json" \
  -b cookies.txt \
  -d '{"reflectionText": "Practiced promises and async/await.", "timeSpent": 45}'
```
