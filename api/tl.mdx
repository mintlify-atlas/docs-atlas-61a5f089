</Info>

## GET /api/tl/dashboard

<Info>Requires authentication — role: `tl`</Info>

Returns all data needed for the TL dashboard: the TL's own profile, all coders in their clan with progress and soft skills, a clan overview summary, and aggregated soft skills averages.

**Response — 200 OK**

<ResponseField name="tl" type="object">
  TL profile.
  <Expandable title="properties">
    <ResponseField name="fullName" type="string">TL's display name.</ResponseField>
    <ResponseField name="clanId" type="string">TL's clan identifier.</ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="coders" type="array">
  All coders in the TL's clan. Each object contains: `id`, `fullName`, `email`, `firstLogin`, `clanId`, `kairoScore`, `averageScore`, `currentWeek`, `autonomy`, `timeManagement`, `problemSolving`, `communication`, `teamwork`, `learningStyle`, `riskLevel`.
</ResponseField>

<ResponseField name="overview" type="object">
  Clan-level summary statistics.
  <Expandable title="properties">
    <ResponseField name="totalCoders" type="number">Total coders in the clan.</ResponseField>
    <ResponseField name="completedOnboarding" type="number">Coders who completed onboarding.</ResponseField>
    <ResponseField name="pendingOnboarding" type="number">Coders who have not completed onboarding.</ResponseField>
    <ResponseField name="highRiskCoders" type="number">Coders with `high` or `critical` risk flags.</ResponseField>
    <ResponseField name="clanAvgScore" type="number">Mean Kairo score across all clan coders.</ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="softSkillsAverage" type="object">
  Clan averages for each soft skill dimension (only from coders who have been assessed).
  <Expandable title="properties">
    <ResponseField name="autonomy" type="number">Average autonomy score.</ResponseField>
    <ResponseField name="time_management" type="number">Average time management score.</ResponseField>
    <ResponseField name="problem_solving" type="number">Average problem solving score.</ResponseField>
    <ResponseField name="communication" type="number">Average communication score.</ResponseField>
    <ResponseField name="teamwork" type="number">Average teamwork score.</ResponseField>
  </Expandable>
</ResponseField>

**Error responses**

| Status | Condition |
|--------|-----------|
| `404 Not Found` | TL user record not found |
| `500 Internal Server Error` | Database error |

**Example**

```bash
curl http://localhost:3000/api/tl/dashboard -b cookies.txt
```

## POST /api/tl/feedback

<Info>Requires authentication — role: `tl`</Info>

Submits a feedback message for a coder. After saving, a real-time SSE notification is dispatched to the coder.

**Request body**

<ParamField body="coderId" type="number" required>
  ID of the coder receiving the feedback.
</ParamField>

<ParamField body="feedbackText" type="string" required>
  Feedback message content.
</ParamField>

<ParamField body="feedbackType" type="string" required>
  Category of the feedback (e.g., `positive`, `improvement`, `urgent`).
</ParamField>

**Response — 201 Created**

<ResponseField name="success" type="boolean">
  `true` on success.
</ResponseField>

<ResponseField name="message" type="string">
  `"Feedback registered"`
</ResponseField>

<ResponseField name="data" type="object">
  The created `tl_feedback` database record.
</ResponseField>

**Error responses**

| Status | Condition |
|--------|-----------|
| `400 Bad Request` | `coderId`, `feedbackText`, or `feedbackType` missing |
| `500 Internal Server Error` | Database error |

**Example**

```bash
curl -X POST http://localhost:3000/api/tl/feedback \
  -H "Content-Type: application/json" \
  -b cookies.txt \
  -d '{"coderId": 12, "feedbackText": "Great progress on async this week!", "feedbackType": "positive"}'
```

## GET /api/tl/submissions

<Info>Requires authentication — role: `tl`</Info>

Returns the 50 most recent exercise submissions from coders in the TL's clan, including the full exercise definition and coder details.

**Response — 200 OK**

<ResponseField name="submissions" type="array">
  Up to 50 submission records ordered by `submittedAt` descending. Each object contains:
  `id`, `codeSubmitted`, `submittedAt`, `tlFeedbackText`, `reviewedAt`, `reviewedBy`,
  `exerciseTitle`, `language`, `difficulty`, `topic`, `solution`, `expectedOutput`,
  `dayNumber`, `coderName`, `coderId`, `planId`.
</ResponseField>

**Error responses**

| Status | Condition |
|--------|-----------|
| `400 Bad Request` | TL has no clan assigned |
| `500 Internal Server Error` | Database error |

**Example**

```bash
curl http://localhost:3000/api/tl/submissions -b cookies.txt
```

## POST /api/tl/submissions/:id/review

<Info>Requires authentication — role: `tl`</Info>

Adds TL feedback to a specific exercise submission. The submission must belong to a coder in the TL's clan. After saving, Kairo points are awarded to the coder and an SSE notification is sent.

**Path parameters**

<ParamField path="id" type="number" required>
  ID of the exercise submission to review.
</ParamField>

**Request body**

<ParamField body="feedbackText" type="string" required>
  Review feedback for the submission. Must not be blank.
</ParamField>

**Response — 200 OK**

<ResponseField name="success" type="boolean">
  `true` on success.
</ResponseField>

<ResponseField name="submission" type="object">
  Updated submission record with `id`, `tlFeedbackText`, and `reviewedAt`.
</ResponseField>

**Error responses**

| Status | Condition |
|--------|-----------|
| `400 Bad Request` | `feedbackText` is blank |
| `404 Not Found` | Submission not found or not in TL's clan |
| `500 Internal Server Error` | Database error |

**Example**

```bash
curl -X POST http://localhost:3000/api/tl/submissions/88/review \
  -H "Content-Type: application/json" \
  -b cookies.txt \
  -d '{"feedbackText": "Correct approach. Consider using Array.reduce instead of a loop."}'
```

## GET /api/tl/coder/:id/score

<Info>Requires authentication — role: `tl`</Info>

Returns a coder's current Kairo score and their 20 most recent score events, with human-readable labels for each event type.

**Path parameters**

<ParamField path="id" type="number" required>
  ID of the coder whose score history to retrieve.
</ParamField>

**Response — 200 OK**

<ResponseField name="coderId" type="number">
  The requested coder's ID.
</ResponseField>

<ResponseField name="currentScore" type="number">
  Current Kairo score.
</ResponseField>

<ResponseField name="history" type="array">
  Up to 20 score events, each with:
  <Expandable title="properties">
    <ResponseField name="label" type="string">Human-readable event label (e.g., `"Día completado"`, `"TL aprobó solución"`).</ResponseField>
    <ResponseField name="points" type="number">Points awarded or deducted for this event.</ResponseField>
    <ResponseField name="createdAt" type="string">ISO 8601 timestamp.</ResponseField>
  </Expandable>
</ResponseField>

**Event type labels**

| Event type | Label |
|------------|-------|
| `day_complete` | Día completado |
| `exercise_submit` | Ejercicio enviado |
| `tl_approved` | TL aprobó solución |
| `plan_complete` | Plan completado (bonus) |
| `inactivity` | Días sin actividad |

**Example**

```bash
curl http://localhost:3000/api/tl/coder/12/score -b cookies.txt
```

## POST /api/tl/resource/upload

<Info>Requires authentication — role: `tl`</Info>

Uploads a PDF resource to Supabase Storage and registers it in the database. After upload, all active coders in the TL's clan receive a real-time notification.

<Warning>
  Only PDF files are accepted. Maximum file size is **20 MB**. The request must be sent as `multipart/form-data`.
</Warning>

**Request body (multipart/form-data)**

<ParamField body="file" type="file" required>
  The PDF file to upload. `Content-Type` must be `application/pdf`.
</ParamField>

<ParamField body="title" type="string" required>
  Display title for the resource.
</ParamField>

<ParamField body="moduleId" type="number">
  Module ID to associate the resource with.
</ParamField>

**Response — 200 OK**

<ResponseField name="success" type="boolean">
  `true` on success.
</ResponseField>

<ResponseField name="message" type="string">
  `"PDF subido correctamente."`
</ResponseField>

<ResponseField name="resourceId" type="number | null">
  Database ID of the new resource record.
</ResponseField>

<ResponseField name="storagePath" type="string">
  Internal path in the Supabase Storage bucket.
</ResponseField>

**Error responses**

| Status | Condition |
|--------|-----------|
| `400 Bad Request` | No file received, file is not a PDF, file exceeds 20 MB, `title` missing, or TL has no clan |
| `500 Internal Server Error` | Storage upload or database error |

**Example**

```bash
curl -X POST http://localhost:3000/api/tl/resource/upload \
  -b cookies.txt \
  -F "file=@/path/to/guide.pdf;type=application/pdf" \
  -F "title=Node.js Best Practices" \
  -F "moduleId=4"
```

## GET /api/tl/resource/list

<Info>Requires authentication — role: `tl`</Info>

Lists all active resources uploaded for the TL's clan, optionally filtered by module.

**Query parameters**

<ParamField query="moduleId" type="number">
  Filter resources by module ID.
</ParamField>

**Response — 200 OK**

<ResponseField name="resources" type="array">
  Array of resource objects ordered by upload date descending. Each contains `id`, `title`, `fileName`, `previewText`, `uploadedAt`, `moduleId`, and `clan`.
</ResponseField>

**Example**

```bash
curl "http://localhost:3000/api/tl/resource/list?moduleId=4" -b cookies.txt
```

## DELETE /api/tl/resource/:resourceId

<Info>Requires authentication — role: `tl`</Info>

Soft-deletes a resource (sets `is_active = false`) and removes the file from Supabase Storage. Only the TL who uploaded the resource can delete it.

**Path parameters**

<ParamField path="resourceId" type="number" required>
  ID of the resource to delete.
</ParamField>

**Response — 200 OK**

<ResponseField name="success" type="boolean">
  `true` on success.
</ResponseField>

**Error responses**

| Status | Condition |
|--------|-----------|
| `404 Not Found` | Resource not found, not in TL's clan, or uploaded by a different TL |
| `500 Internal Server Error` | Database or storage error |

**Example**

```bash
curl -X DELETE http://localhost:3000/api/tl/resource/12 -b cookies.txt
```

## GET /api/tl/ranking

<Info>Requires authentication — role: `tl`</Info>

Returns two ranked lists: all coders in the TL's clan ordered by Kairo score, and the global top 10 coders across all clans.

**Response — 200 OK**

<ResponseField name="clan" type="string">
  The TL's clan identifier.
</ResponseField>

<ResponseField name="clanRanking" type="array">
  All active coders in the clan ranked by `kairoScore` descending. Each item has `id`, `fullName`, `kairoScore`, `clan`, and `rank`.
</ResponseField>

<ResponseField name="globalRanking" type="array">
  Top 10 active coders globally ranked by `kairoScore` descending. Each item has `id`, `fullName`, `kairoScore`, `clan`, and `rank`.
</ResponseField>

**Example**

```bash
curl http://localhost:3000/api/tl/ranking -b cookies.txt
```
